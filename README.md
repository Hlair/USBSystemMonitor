PC Resource Monitor with ESP32 External Display
Апаратно-програмний комплекс для моніторингу системних ресурсів ПК у реальному часі з виведенням телеметрії на зовнішній TFT-дисплей на базі мікроконтролера ESP32.

Проєкт створено для розвантаження основного монітора від системних оверлеїв під час геймінгу, 3D-рендерингу або компіляції важких програм.

Десктопний застосунок (Хост)
Низькорівневий збір метрик: Використання нативних API Windows (PDH, WinAPI, DXGI) для точного зчитування навантаження на CPU, RAM, Мережу та Диски.

Підтримка NVIDIA NVML: Динамічне завантаження бібліотеки nvml.dll для точного визначення навантаження графічного ядра NVIDIA (з автоматичним фолбеком на DXGI для AMD/Intel).

Ультранизьке споживання ресурсів: Написано на C++ з використанням Dear ImGui. Програма споживає ~50 МБ RAM та менше 0.5% CPU.

Багатопотокова архітектура: Розділені потоки рендерингу GUI та фонового опитування/передачі даних через UART (std::thread, std::mutex).

Plug-and-Play (Автопідключення): Розумний менеджер COM-портів з алгоритмом "рукостискання" (Handshake) — програма сама знаходить підключену плату ESP32 без ручного налаштування.

Інтеграція з Windows: Підтримка системного трею (ShellAPI) та нативної темної теми вікна (DWM API).

Вбудоване ПЗ (ESP32)
Двоядерна обробка (FreeRTOS): Ядро 0: Безперервний асинхронний прийом бінарних пакетів по UART, розрахунок контрольної суми (XOR).

Ядро 1: Графічний рендеринг без впливу на втрату пакетів.

Гібридний графічний рушій: Використання LVGL (Light and Versatile Graphics Library) для згладжених (anti-aliased) віджетів та анімацій (радіальні дуги, стрілкові спідометри).

Використання Adafruit_GFX з кастомним алгоритмом «дельта-відмальовки» для миттєвого оновлення текстових метрик без мерехтіння дисплея (tearing) та зайвого навантаження на шину SPI.

Дистанційне керування: Перемикання графічних режимів відображення безпосередньо з ПК.


🛠️ Стек технологій
PC (Windows):

Мова: C++

UI: Dear ImGui, GLFW, OpenGL 3

API: Windows API, PDH (Performance Data Helper), DXGI, DWMAPI.

ESP32 (Device):

Мова: C++ (Arduino Framework / PlatformIO)

ОС: FreeRTOS

Графіка: LVGL, Adafruit_GFX, Adafruit_ST7789

Інтерфейси: UART (Hardware Serial), SPI.

🚀 Встановлення та збірка
1. Збірка програми для ПК (Visual Studio)
Склонуйте репозиторій.

Відкрийте рішення (.sln) у Visual Studio 2019/2022.

Переконайтеся, що підключені бібліотеки GLFW та ImGui.

У властивостях лінкера (Linker -> Input) перевірте наявність: dwmapi.lib, pdh.lib, shell32.lib.

Зберіть проєкт у конфігурації Release / x64.

2. Прошивка ESP32 (Arduino IDE)
Підключіть ESP32 до ПК.

Встановіть необхідні бібліотеки через Library Manager:

Adafruit GFX Library

Adafruit ST7789 Library

lvgl (версії 8.x)

Відкрийте файл .ino, оберіть плату "ESP32 Dev Module" та завантажте прошивку.

(Схема підключення дисплея до пінів SPI вказана у заголовку файлу .ino)

🔌 Використання
Підключіть плату ESP32 до комп'ютера за допомогою USB-кабелю.

Запустіть скомпільований .exe файл.

Програма автоматично знайде пристрій і почне трансляцію телеметрії.

Для перемикання екранів використовуйте відповідні кнопки на панелі керування (Control Center) застосунку.

При згортанні вікна програма сховається в системний трей для економії місця.

📝 Структура пакету UART (Протокол)
Для комунікації розроблено кастомний бінарний протокол. Кожен пакет має байт 0xAA, команду, корисне навантаження та байт XOR-контрольної суми для валідації цілісності:

#pragma pack(push, 1)
struct PacketHeader {
    uint8_t magic = 0xAA;
    CommandType command;
    uint16_t payload_length;
};
// ... Metrics Payload ...
#pragma pack(pop)
